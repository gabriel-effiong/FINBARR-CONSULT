🚀 GET STARTED IN 3 STEPS
═══════════════════════════════════════════════════════════════════════════════

Step 1: Install Dependencies (1 minute)
────────────────────────────────────────────────────────────────────────────────
  $ cd backend
  $ npm install

Step 2: Configure Environment (2 minutes)
────────────────────────────────────────────────────────────────────────────────
  $ cp .env.example .env
  
  Edit .env and add:
  - MongoDB URI (get free at mongodb.com/cloud/atlas)
  - Gmail credentials (or other email service)
  
  Example:
  MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/finbarr
  SMTP_USER=your-email@gmail.com
  SMTP_PASSWORD=xxxx xxxx xxxx xxxx

Step 3: Start Server (1 minute)
────────────────────────────────────────────────────────────────────────────────
  $ npm run dev
  
  You should see:
  ✅ Server running on port 5000
  📧 Frontend URL: https://finbarr-consult.vercel.app
  🔐 Environment: development

═══════════════════════════════════════════════════════════════════════════════
⚠️  CRITICAL CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════

Before going to production:

□ Change Admin Password
  Update ADMIN_PASSWORD in .env to a strong password

□ Generate Strong JWT Secret
  Run: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  Copy output to JWT_SECRET in .env

□ Setup MongoDB
  Get connection string from mongodb.com/cloud/atlas

□ Configure Email
  Get app password from myaccount.google.com/apppasswords

□ Update Frontend API URL
  Change API_BASE_URL in api-integration.js to production URL

□ Enable HTTPS
  Only use HTTPS URLs in production

□ Setup Backups
  Enable MongoDB backups
